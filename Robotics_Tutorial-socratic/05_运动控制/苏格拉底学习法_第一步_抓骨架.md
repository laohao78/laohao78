一句话：运动控制不是"让机器人动起来"，而是把一个受动力学、接触、硬件和实时性限制的多体系统，变成一个稳定、可解释、可迁移的闭环行为。

---

<h2><span style="color: #e2b714">①</span> 多体动力学 —— <span style="color: #888">机器人不是点，是受约束的物理系统</span></h2>

运动控制的第一问不是"用什么算法"，而是：

> <span style="color: #e2b714; font-size: 1.05em"><b>这个机器人有多少自由度？哪些自由度有电机？哪些自由度靠接触力间接控制？</b></span>

固定基座机械臂最干净：基座不动，每个关节通常都有电机，是全驱动系统。

$$M(q)\ddot{q} + h(q,\dot{q}) = \tau$$

足式和人形突然复杂：基座在空中，是 6 个无驱动自由度；机器人能不能加速、转身、站稳，取决于地面对脚的接触力。

$$M(q)\ddot{q} + h(q,\dot{q}) = S^T\tau + J_c^T\lambda$$

复合机器人再加一层耦合：四足走路时机械臂一甩，臂的反力矩会扰动基座稳定性。移动操作也一样，底盘位姿决定机械臂可达空间，机械臂动作又反过来改变整体质心。

| 系统 | 关键变化 | 你该问的问题 |
|------|---------|-------------|
| 机械臂 | 固定基座，全驱动 | 末端怎么到？关节限位/碰撞怎么避？ |
| 足式 | 浮动基座，接触支撑 | 脚踩哪里？接触力够不够？会不会滑？ |
| 四足+臂 | 浮基 + 操作耦合 | 手在动时腿怎么补偿？臂反力矩怎么进 WBC？ |
| 人形 | 30+ DOF，小支撑面 | 高维全身动作如何保持平衡？ |

**对应材料**：机械臂 M01，足式 30/50，复合 20，仿真 S01。共同工具是 Pinocchio/MuJoCo 这类多体动力学库。

---

<h2><span style="color: #e74c3c">②</span> 约束最优控制 —— <span style="color: #888">控制就是带物理约束的决策</span></h2>

从机械臂轨迹优化，到四足 OCS2 MPC，到人形 WBC，本质都在做同一件事：

$$\min_{x,u,\lambda} \sum_k \ell(x_k,u_k,\lambda_k) \quad \text{s.t.} \quad x_{k+1}=f(x_k,u_k,\lambda_k),\ g(x_k,u_k,\lambda_k)\le 0$$

区别只在于：状态维度、约束种类、求解速度、是否允许学习模块参与。

| 方法 | 典型位置 | 核心问题 |
|------|---------|---------|
| IK / 差分 IK | 机械臂入门 | 末端误差如何变成关节速度？ |
| QP / HQP / TSID | WBC / 力控 | 多任务优先级和约束如何同时满足？ |
| DDP / iLQR / ProxDDP | 轨迹优化 | 非线性动力学下怎样快速更新轨迹？ |
| MPC / SQP-RTI | 足式/复合实时控制 | 每个控制周期只算一点，如何保持闭环稳定？ |
| TOPP / Ruckig | 时间参数化 | 几何路径如何变成满足速度/加速度限制的时间轨迹？ |

<p style="border-left: 3px solid #e74c3c; padding-left: 16px">
SLAM 里你熟悉的是"最小化重投影误差/点面误差"。运动控制里多了一件事：<b>约束是主角</b>。关节限位、摩擦锥、ZMP、接触互补、碰撞距离、实时延迟，都会进入优化问题。
</p>

**对应材料**：机械臂 M05/M08/M10，足式 60/100/110，复合 30/70/80，仿真 S05。

---

<h2><span style="color: #5dade2">③</span> 接触与力 —— <span style="color: #888">从"不撞"升级为"怎么碰"</span></h2>

移动机器人规控里，障碍物通常是几何对象：不要撞上墙、车、人。

运动控制里，接触不是例外，而是能力来源：

- 机械臂要靠接触完成插入、擦拭、抓取、装配。
- 四足要靠脚地接触产生运动。
- 人形要靠脚底支撑多边形保持平衡。
- 移动操作要同时处理轮地、脚地、夹爪物体、物体环境的多重接触。

接触有三条底层规则：

| 规则 | 含义 | 失败表现 |
|------|------|---------|
| Signorini | 没接触就不能有法向力 | 脚离地还在"推地" |
| Coulomb 摩擦 | 切向力不能超过摩擦锥 | 脚打滑、夹爪滑脱 |
| 互补性 | 接触距离和接触力不能同时为正 | 既穿透又不受力的仿真假象 |

这就是为什么阻抗/导纳控制、WBC、TSID、接触隐式优化、力敏感 RL 会在运控里变成核心主题。

> <span style="color: #5dade2; font-size: 1.05em"><b>几何规划问"能不能到达"，接触控制问"用多大的力、以什么柔顺性、在什么约束下到达"。</b></span>

**对应材料**：机械臂 F01-F10，足式 80/90，复合 170/210/250，仿真 S04。

<h2><span style="color: #27ae60">④</span> 分层实时闭环 —— <span style="color: #888">慢脑子 + 快反射</span></h2>

运动控制系统永远是分层的。不是因为研究者喜欢画架构图，而是因为不同层的时间尺度差太多。

| 层 | 频率 | 典型算法 | 解决的问题 |
|----|------|---------|-----------|
| 任务层 | 0.1-10 Hz | BT / FSM / VLA / MTC | 要做什么？失败后怎么办？ |
| 轨迹层 | 10-100 Hz | MPC / DDP / MoveIt2 | 接下来几秒怎么动？ |
| 全身控制层 | 100-500 Hz | WBC / TSID / QP | 任务如何变成关节加速度/力矩？ |
| 伺服层 | 500-1000+ Hz | PD / 阻抗 / ros2_control | 电机实际输出什么？ |

这也是为什么 `05_运动控制` 大纲反复强调 PREEMPT_RT、无堆分配、SCHED_FIFO、lock-free、`EIGEN_RUNTIME_NO_MALLOC`。再漂亮的 MPC，如果控制循环随机卡 20ms，真机照样抖。

<p style="border-left: 3px solid #27ae60; padding-left: 16px">
运动控制里的"实时"不是"程序跑得快"。它是<b>最坏情况延迟可控</b>。平均 0.2ms、偶尔 50ms 的控制器，比稳定 1ms 的控制器更危险。
</p>

**对应材料**：机械臂 M11/M12/M13/M14，足式 90/110/170/180/240，复合 100/120/170，ros2_control 交叉章节。

<h2><span style="color: #c586c0">⑤</span> 仿真学习迁移 —— <span style="color: #888">用仿真和数据补足模型控制的边界</span></h2>

纯模型控制的强项是可解释、安全、能诊断；弱项是模型不完美、接触复杂、感知输入难建模。RL/VLA/动作模仿的强项是表达能力和数据驱动泛化；弱项是安全性、可解释性、分布外失败。

所以 2025-2026 的主线不是"模型控制被学习替代"，而是：

<div style="font-family: monospace; line-height: 1.8; padding: 8px 16px; background: rgba(255,255,255,0.03); border-radius: 4px">
<span style="color: #e2b714">模型控制</span> 提供物理结构和安全边界<br>
<span style="color: #27ae60">RL / 动作模仿</span> 提供高维技能和感知反应<br>
<span style="color: #5dade2">仿真器</span> 提供大规模试错和 sim-to-real 验证<br>
<span style="color: #c586c0">VLA</span> 提供语义意图和开放世界先验<br>
</div>

跨方向方法论：

| 方法 | 作用 |
|------|------|
| Domain Randomization | 让策略见过更多物理参数 |
| System Identification | 缩小仿真和真机参数差距 |
| Delta-Action Residual | 仿真策略打底，真机小残差补偿 |
| Sim2Sim 验证 | IsaacLab/MuJoCo/Genesis 交叉检查 |
| 可微分仿真/可微 MPC | 让梯度穿过物理或 MPC 层 |

**对应材料**：足式 190/200/210，复合 140/230/240/260/270，仿真 S03-S05，MJLab 工程手册，动作模仿理论。

<h2><span style="color: #e2b714">一句 Skeleton Summary</span></h2>

> <span style="color: #e2b714; font-size: 1.05em"><b>运动控制 = 先把机器人建成一个受约束的多体动力学系统，再用 MPC/WBC/RL 等方法在接触、力、实时性和硬件限制下求闭环动作；仿真与学习不是替代物理，而是让物理控制在高维感知、复杂接触和真机迁移中可用。</b></span>

<h2>和你已学的 <span style="color: #5dade2">SLAM</span> / <span style="color: #27ae60">移动规控</span> 的关系</h2>

| 你已学过 | 在 05_运动控制 中变成 |
|---------|----------------------|
| SLAM 位姿/地图 | MPC/WBC 的状态与环境输入 |
| ESDF/点云 | 碰撞/SDF/高程图/Perceptive MPC 约束 |
| Nav2 层级架构 | BT/MoveIt2/ros2_control/控制分层 |
| MPC/MPPI 最优决策 | 足式/复合/机械臂的约束最优控制 |
| 不确定性规划 | sim-to-real、接触鲁棒性、地图不确定性进入代价 |
| 博弈/多机协作 | 多机器人 LocoMani、协作搬运、分布式 MPC |

你已经有"感知到决策"的背景。进入运动控制后，关键升级是：把"机器人在地图里怎么走"升级为"机器人作为一个真实物理系统，怎样在接触和硬件限制下执行动作"。
