<h2><span style="color: #e74c3c">争议 1</span>：显式 MPC/WBC vs 端到端 RL</h2>

<span style="color: #e74c3c"><b>全身控制应该由可解释的动力学优化器求解，还是由神经网络直接输出动作？</b></span>

<h3>两个阵营</h3>

| | <span style="color: #e2b714">显式 MPC/WBC 派</span> | <span style="color: #27ae60">端到端 RL 派</span> |
|---|---|---|
| <b>信条</b> | 物理约束必须显式编码 | 高维接触太复杂，让数据学 |
| <b>代表</b> | OCS2 legged_robot, qm_control, TSID, acados | legged_gym, Deep WBC, FALCON, GR00T-WBC |
| <b>强项</b> | 可解释、约束清楚、能诊断失败 | 高维、低延迟、感知和动作可端到端 |
| <b>弱项</b> | 调参重，模型误差敏感，30+ DOF 维度压力大 | 安全保证弱，分布外失败不可预测 |
| <b>典型平台</b> | 四足、四足+臂、工业机械臂 | parkour、人形、复杂操作策略 |

<h3>真正在吵什么</h3>

不是"优化器好还是网络好"，而是对失败风险的排序不同。

- <span style="color: #e2b714">MPC/WBC 派</span>：最怕不可诊断的失败。宁愿控制器笨一点，也要知道约束为什么没满足。
- <span style="color: #27ae60">RL 派</span>：最怕人工模型表达不了真实任务。宁愿靠训练覆盖复杂场景，也不想手工写出所有代价和约束。

<p style="border-left: 3px solid #e74c3c; padding-left: 16px">
真正的前沿答案不是二选一，而是混合：MPC/WBC 给安全边界，RL 学残差、参数、感知策略或高层技能。RAMBO、MPC-Net、Cafe-MPC、Teacher-Student 都是这条中间路线。
</p>

<h3><span style="color: #e2b714">关联第一步模型</span></h3>

模型②（约束最优控制）、模型③（接触与力）、模型⑤（仿真学习迁移）。这是运控领域最核心的范式争议。

---

<h2><span style="color: #5dade2">争议 2</span>：模块化松耦合 vs 全身联合优化</h2>

<span style="color: #5dade2"><b>底盘、机械臂、腿、手、任务规划应该分开做，还是放进同一个全身优化问题？</b></span>

<h3>两个阵营</h3>

| | <span style="color: #e2b714">模块化松耦合</span> | <span style="color: #5dade2">全身联合优化</span> |
|---|---|---|
| <b>信条</b> | 拆开才可维护、可测试、可替换 | 耦合强时拆开会丢掉决定性信息 |
| <b>代表</b> | Nav2 + MoveIt2 + BT, MoveIt2 + ros2_control | OCS2 mobile_manipulator, WBC/TSID, loco-mani MPC |
| <b>适用</b> | 底盘停稳再抓、工业 pick-and-place、弱耦合任务 | 行走中操作、四足+臂、人形、动态抓取 |
| <b>优势</b> | 工程成熟，调试边界清楚 | 能处理底盘-臂-接触-平衡强耦合 |
| <b>风险</b> | 强耦合时出现不可行或动作保守 | 模型大、求解慢、调参复杂 |

<h3>真正在吵什么</h3>

这是运动控制版的"路径-速度解耦 vs 时空联合"。区别是这里耦合的不只是路径和速度，而是身体部件之间的动力学。

一个移动机械臂如果先 Nav2 到物体旁边，再 MoveIt2 规划抓取，很多仓储任务足够好。但四足走路时用机械臂推门，臂的力会改变腿的接触力分配；这时"先走再抓"的接口就太粗了。

> <span style="color: #5dade2; font-size: 1.05em"><b>弱耦合时拆开是工程智慧，强耦合时拆开是问题来源。</b></span>

<h3><span style="color: #e2b714">关联第一步模型</span></h3>

模型①（多体动力学）和模型④（分层实时闭环）。判断能不能模块化，先看动力学和接触是否强耦合。

---

<h2><span style="color: #27ae60">争议 3</span>：可微物理梯度 vs 大规模采样训练</h2>

<span style="color: #27ae60"><b>应该让梯度穿过物理引擎/MPC，还是用海量并行仿真和 RL 采样硬推？</b></span>

<h3>两个阵营</h3>

| | <span style="color: #e2b714">可微物理 / 可微 MPC</span> | <span style="color: #27ae60">大规模采样 / RL</span> |
|---|---|---|
| <b>信条</b> | 梯度给出方向，样本效率更高 | 接触太非光滑，采样更鲁棒 |
| <b>代表</b> | MJX, SHAC, leap-c, acados differentiable NMPC | IsaacLab, MuJoCo Playground, mjlab, PPO/AMP |
| <b>强项</b> | 低接触任务、参数优化、MPC 层学习 | 高接触、高维、人形/四足大规模训练 |
| <b>弱项</b> | 接触梯度可能有偏甚至方向错 | 样本多、奖励工程重、解释性弱 |
| <b>判断标准</b> | 梯度质量是否可信 | 并行仿真速度与 sim-to-real 是否够好 |

<h3>真正在吵什么</h3>

可微派相信：如果能把物理写成可微函数，就能更聪明地训练控制器。采样派反击：接触不是光滑函数，脚刚碰地、夹爪刚接触物体时，梯度本身就可能误导。

MuJoCo/JAX/MJX、Warp、Newton、Genesis 的竞争表面上是仿真器生态竞争，底层其实是两种学习哲学：

- <span style="color: #e2b714">用更好的梯度减少试错</span>
- <span style="color: #27ae60">用更快的仿真承担试错</span>

<p style="border-left: 3px solid #27ae60; padding-left: 16px">
判断标准不是"哪个更先进"，而是你的任务接触是否密集。低接触、连续动力学任务更适合可微；脚地、手物、物体环境多接触任务，采样和 DR 仍然很难被替代。
</p>

<h3><span style="color: #e2b714">关联第一步模型</span></h3>

模型③（接触与力）和模型⑤（仿真学习迁移）。接触非光滑性是这个争议的根。

---

<h2><span style="color: #c586c0">争议 4</span>：VLA/Foundation Model 能不能直接接管控制</h2>

<span style="color: #c586c0"><b>机器人基础模型会不会让 MPC/WBC/MoveIt2 这些传统控制栈失去意义？</b></span>

<h3>两个阵营</h3>

| | <span style="color: #e2b714">控制栈兜底派</span> | <span style="color: #c586c0">Foundation Policy 派</span> |
|---|---|---|
| <b>信条</b> | VLA 懂语义，不等于懂力和约束 | 大规模数据会学到跨机器人技能 |
| <b>代表</b> | VLA + MPC safety filter, VLA + WBC, MoveIt2 + policy | pi0/pi0.5, SpatialVLA, GR00T, OpenVLA |
| <b>强项</b> | 安全边界明确，能接工业控制器 | 开放世界泛化、语义理解、多任务 |
| <b>弱项</b> | 接口设计复杂，上限受手工控制栈限制 | 精度弱、安全弱、力觉/触觉处理仍不足 |
| <b>典型接口</b> | VLA 输出目标/原语，MPC/WBC 执行 | 模型直接输出动作序列 |

<h3>真正在吵什么</h3>

VLA 最擅长的是"理解任务"：哪个杯子、什么动作、先后顺序、开放世界语义。运动控制最难的是"物理执行"：关节限位、接触力、摩擦、平衡、实时延迟。

所以目前更稳的架构是：

<div style="font-family: monospace; line-height: 1.8; padding: 8px 16px; background: rgba(255,255,255,0.03); border-radius: 4px">
VLA / LLM：选择任务、目标、操作原语<br>
MoveIt2 / MPC：检查几何、动力学、碰撞、约束<br>
WBC / 阻抗：处理接触、力、平衡<br>
ros2_control：把动作送到硬件
</div>

<p style="border-left: 3px solid #c586c0; padding-left: 16px">
VLA 可以成为"脑"，但短期内不能裸连电机。尤其在机械臂力控、四足+臂、人形操作里，底层安全控制不是可选项。
</p>

<h3><span style="color: #e2b714">关联第一步模型</span></h3>

模型④（分层实时闭环）和模型⑤（仿真学习迁移）。VLA 的正确位置首先是层级问题，其次才是模型能力问题。

---

<h2><span style="color: #e2b714">立场总结</span></h2>

| 争议 | 当前更稳的立场 |
|------|---------------|
| 显式 MPC/WBC vs 端到端 RL | 中短期混合范式最强：MPC/WBC 兜底，RL 学高维技能或残差 |
| 模块化松耦合 vs 全身联合优化 | 弱耦合用模块化，强耦合用联合优化；先判断耦合强度 |
| 可微物理 vs 大规模采样 | 低接触任务偏可微，高接触高维任务偏采样+DR |
| VLA 能否接管控制 | VLA 做语义和目标，控制栈做约束与执行；直接接管只适合低风险研究原型 |

<h2>从<span style="color: #5dade2">你</span>的视角看这四场争议</h2>

你已经学过 `03_SLAM` 和 `04_移动机器人规控`。这意味着你不是从零进入运动控制，而是从"感知-导航-规划"进入"物理执行"。

| 争议 | 和你的关系 |
|------|-----------|
| 显式 MPC/WBC vs RL | 你在移动规控里已经理解 MPC/MPPI。进入足式/复合后，要补 WBC、接触约束和 RL 部署 |
| 模块化 vs 全身联合 | 你熟悉 Nav2 模块化。移动操作一开始可以沿用 Nav2+MoveIt2，但四足+臂会逼你理解联合动力学 |
| 可微物理 vs 采样训练 | 你的 SLAM 优化背景适合理解可微 MPC，但真机落地还要学 DR/SysID/sim2sim |
| VLA 是否接管控制 | 你做语义导航/LightMap 的经验适合做 VLA 高层接口，但底层必须由 MPC/WBC/安全滤波兜底 |
