<h2>方向一：<span style="color: #5dade2">SLAM 不确定性感知的 Perceptive MPC</span> <span style="color: #e2b714">S 级 ← 最推荐</span></h2>

<h3>问题</h3>

足式/轮足机器人在复杂地形上走，不只是需要高程图，还需要知道这张高程图有多可信。传统做法常把 SLAM 输出当成确定地图：高程、坡度、障碍物直接进入 MPC。问题是，点云稀疏、遮挡、动态物体、地面反光都会让地图局部不确定。

如果 MPC 不知道地图哪里不确定，它会把"看不清"误当成"能走"。

<h3>核心思路</h3>

把 SLAM/高程图的不确定性传给 MPC：

| SLAM 输出 | 控制层用法 |
|----------|-----------|
| 高程均值 | 足端目标高度、身体姿态 |
| 高程方差 | 落脚风险、速度上限、保守度 |
| ESDF 距离 | 碰撞约束 |
| 地图置信度 | MPC 代价权重自适应 |
| 语义地形类别 | 摩擦系数/可通行性先验 |

这不是"给 costmap 加个权重"，而是把 SLAM 的信念传播到控制器的约束和风险预算里。

<h3>为什么适合你</h3>

- 你已学 SLAM，理解地图和状态估计的不确定性。
- 你已学移动规控，知道 MPC/MPPI 的决策语言。
- 你进入运动控制后，只需要补足式 230 Perceptive MPC、WBC、接触力，就能形成闭环。

<h3>可落地题目</h3>

<blockquote style="border-left:4px solid #e2b714; padding-left: 16px">
<b>Uncertainty-aware Perceptive MPC for Legged Navigation</b><br>
将 elevation map 的均值/方差/语义类别转化为 MPC 落脚风险和速度调度，在稀疏点云、遮挡和未知地形中比确定性高程图更稳。
</blockquote>

<h2>方向二：<span style="color: #27ae60">语义移动操作的安全执行层</span> <span style="color: #e2b714">S/A 级</span></h2>

<h3>问题</h3>

你之前的语义导航/LightMap 能让机器人理解"门、电梯、房间、目标区域"。进入 05 后，可以把这个能力接到移动操作上：机器人不只是走到目标，还要开门、取物、按按钮、递送。

但 VLA/语义模块输出的是高层意图，不保证运动安全。

<h3>核心思路</h3>

构建一个 VLA/语义决策 + 传统控制兜底的移动操作栈：

<pre style="background:#1e1e1e; color:#d4d4d4; padding:16px; border-radius:8px; line-height:1.8;">
<span style="color: #5dade2">语义地图 / VLA</span>
       │  输出目标、物体、操作原语
       ▼
<span style="color: #e2b714">BT / MTC 任务编排</span>
       │  Navigate -> Align -> Reach -> Grasp -> Place
       ▼
<span style="color: #27ae60">Nav2 + MoveIt2 / OCS2 mobile_manipulator</span>
       │  几何、碰撞、底盘-臂协调
       ▼
<span style="color: #e74c3c">MPC / WBC / 阻抗安全执行</span>
       │  关节限位、力控、碰撞、接触安全
       ▼
<span style="color: #888">ros2_control / 真机</span>
</pre>

<h3>研究增量在哪里</h3>

不要只做"把 VLA 接到机械臂上"。真正有价值的是定义接口：

- VLA 输出末端位姿还是操作原语？
- 语义不确定性如何变成规划风险？
- MoveIt2 失败时，BT 如何回退到重新观察/换抓取姿态？
- 接触任务中，何时从位置控制切到阻抗/导纳控制？

<h3>与你已有工作的关系</h3>

LightMap/OPEN 可以提供语义目标和探索策略；05_运动控制 提供执行层。这个方向自然连接 `03_SLAM`、`04_移动机器人规控`、`05_运动控制`。

---

<h2>方向三：<span style="color: #e74c3c">四足+臂 Loco-Manipulation 的混合控制</span> <span style="color: #888">A 级</span></h2>

<h3>问题</h3>

四足+臂是运动控制里很强的研究方向：机器人既要走，又要操作。核心难点是臂反力矩会扰动腿部平衡，末端任务和身体稳定必须同时满足。

<h3>核心思路</h3>

从三种架构里选一条：

| 架构 | 代表 | 适合程度 |
|------|------|---------|
| MPC+WBC | qm_control | 可解释、安全，适合你先学 |
| 端到端 RL | Deep WBC | 上限高，但安全和数据要求高 |
| MPC+RL 混合 | RAMBO | 前沿强，但实现复杂 |

最稳路线：先复现 MPC+WBC，再加 RL 残差或 VLA 高层目标。

<h3>风险</h3>

这个方向硬件/仿真门槛更高，需要 Go2+Z1 或类似平台、IsaacLab/MuJoCo 训练环境、WBC/QP 实时调试能力。适合作为你补完基础后的长期方向，而不是第一篇论文就硬冲。

---

<h2>方向四：<span style="color: #c586c0">VLA + MPC Safety Filter</span> <span style="color: #888">A/B 级</span></h2>

<h3>问题</h3>

VLA 模型擅长开放世界语义，但不擅长硬约束和接触力。最自然的研究问题是：如何让 VLA 负责"想做什么"，MPC/WBC 负责"能不能安全做"。

<h3>核心思路</h3>

VLA 输出候选动作或末端目标，MPC safety filter 检查：

- 碰撞距离
- 关节限位
- 力矩/速度限制
- 末端可达性
- 足式平台的 ZMP/接触稳定性
- 动作失败后的回退策略

<h3>为什么不是最推荐</h3>

这个方向热，但容易陷入大模型工程：数据、算力、模型部署会吞掉你的时间。更好的姿势是把它作为方向二的高层模块，而不是把研究重心放在训练 VLA 本身。

---

<h2>方向五：<span style="color: #888">纯人形 Foundation Control / 大规模 RL</span> <span style="color: #e74c3c">C 级（暂时别碰）</span></h2>

<h3>不推荐的原因</h3>

- 30+ DOF 人形训练需要大量 GPU、动捕数据、仿真工程和真机安全资源。
- GR00T-WBC、Figure、Physical Intelligence、NVIDIA 等团队有极强资源优势。
- 你已有的 SLAM/语义导航优势在这个方向里用不上太多。
- 很容易从"研究问题"变成"调 reward、找数据、堆算力"。

更好的策略：先把感知-控制闭环做扎实。等你有 Perceptive MPC / 移动操作安全执行的系统基础后，再把人形作为平台扩展。

---

<h2>统一方向：<span style="color: #e2b714">Perception-aware Whole-body Control</span></h2>

可以把方向一、二、三合成一条统一主线：

<blockquote style="border-left:4px solid #e2b714; padding-left: 16px">
<b>地图不是给规划器看的静态背景，而是控制器的风险输入。</b><br>
机器人同时维护几何状态、语义信念和物理约束，在 MPC/WBC 中把感知不确定性、接触安全和任务目标一起考虑。
</blockquote>

<pre style="background:#1e1e1e; color:#d4d4d4; padding:16px; border-radius:8px; line-height:1.8;">
<span style="color: #5dade2">SLAM / 语义地图</span>
       │  位姿、点云、高程、ESDF、语义置信度
       ▼
<span style="color: #e2b714">风险建模</span>
       │  地图不确定性 -> MPC 权重 / chance constraint / 安全裕度
       ▼
<span style="color: #27ae60">MPC / WBC</span>
       │  动力学、接触、力矩、碰撞、任务目标
       ▼
<span style="color: #c586c0">学习模块</span>
       │  VLA 高层意图 / RL 残差 / 动作模仿策略
       ▼
<span style="color: #e74c3c">实时执行与验证</span>
       │  ros2_control / sim2sim / DR / SysID / 真机安全
</pre>

这条线的优势是：你已有的 SLAM 和移动规控不是背景材料，而是研究核心的一部分。
