<h2><span style="color: #e2b714">①</span> 规划 = 最优决策 —— <span style="color: #888">通用语言</span></h2>

MPC、iLQR、MPPI、MINCO、TEB、CEM、Diffusion Planner、A*、Bellman DP——不管外表差异多大，**本质都是同一个问题：在约束下求最优决策。** 优化（连续）是最常见的实现方式，但不是唯一方式——图搜索和动态规划也是求解手段。

$$\min_{u_{0:N-1}} \sum_{k=0}^{N-1} \ell(x_k, u_k) + \ell_f(x_N) \quad \text{s.t.} \quad x_{k+1}=f(x_k, u_k), \ x_k \in \mathcal{X}, \ u_k \in \mathcal{U}$$

> <span style="color: #e2b714; font-size: 1.05em"><b>代价函数 + 动力学约束 + 环境约束 → 求最优决策。差异在参数化方式和求解器——优化（MPC/iLQR/MINCO）、搜索（A*/Dijkstra/Hybrid A*）、动态规划（Bellman DP）、采样（MPPI/CEM）、学习（RL/Diffusion）。</b></span>

T0 原文："大家只是参数化不同，优化器不同，本质全是同一个数学问题。" S5：所有方法——包括 RL——共享同一个祖先：**Bellman 最优性原理**。

| 方法 | 求解范式 | 何时用 |
|------|---------|--------|
| A* / Dijkstra / Hybrid A* | 图搜索 | 离散状态空间，最短路径 |
| Bellman DP | 递归最优性 | 有最优子结构，维度低 |
| MPC / iLQR | 连续优化（Riccati / SQP） | 可微动力学 + 可微代价 |
| MPPI / CEM | 采样 + 加权 | 不可微代价 / 黑箱动力学 |
| MINCO | 连续优化（L-BFGS） | 轨迹需满足走廊约束 |
| Diffusion / RL | 学习 | 代价难以手工设计 |

<span style="color: #27ae60">这条没有争议。</span> MPPI 总论自述："MPPI 不是梯度式 MPC 的替代品，是互补品。"

---

<h2><span style="color: #e74c3c">②</span> 时空耦合 —— <span style="color: #e74c3c">放松"空间时间可分离"</span></h2>

经典规划栈的隐含假设：**空间和时间可以分开处理**——先决定走哪条路（空间），再决定走多快（时间）。这叫 PVD（路径-速度解耦）。

T0 原文："PVD 的'先后分开'本质是一种**用问题分解换求解效率的工程妥协**——这个妥协在弱耦合时几乎无损，在强耦合时却会丢掉决定性的信息。"

<span style="color: #e2b714">注意：不是"时空不可分"——很多场景就是可分的。</span>工业界主流系统（Apollo、Autoware、Nav2）仍然是空间规划 + 速度规划解耦——因为在大多数场景下解耦够用且实时性更好。时空联合是处理**强耦合场景**的工具（加塞、无保护左转、无人机穿树林），而非对所有场景的普适要求。

| 弱耦合（解耦够用） | 强耦合（解耦失效） |
|---|---|
| 高速公路巡航：路基本是直的 | 加塞：让/抢/变道三个选择互相纠缠 |
| 稀疏交通的城区 | 无保护左转：路径和速度必须同时决定 |
| 静态障碍绕行 | 传送带抓取：抓取点空间位置与到达时刻绑定 |

**失效根因**：动态障碍是**时空物体**——在特定时刻占据特定位置。躲开它需要协调"我在那个时刻走到哪里"，这同时牵涉路径和速度。分两步就破坏了协调。

> <span style="color: #e74c3c; font-size: 1.05em"><b>面对一个规划场景，先问"路径和速度耦合强不强"——弱则放心解耦（快、够用），强则必须联合（慢、但正确）。</b></span>

**源材料**：T0-T6（10_时空规划，8 文件，~16 周）。方法链：Frenet/ST 图 → 时空走廊 → CILQR/TEB/MINCO → Apollo 工业架构 → 多智能体协调 → 扩散式规划。

---

<h2><span style="color: #5dade2">③</span> 不确定性有结构 —— <span style="color: #5dade2">放松"世界是确定的"</span></h2>

U0 原文："不是'把名义 MPC 调得更保守一点'（盲目加大安全距离会让机器人寸步难行），而是'**为不确定性显式建模，并据此分配恰到好处的保守度**'。"

**第一刀：认知 vs 偶然。** 认知不确定性可以靠"多看一眼"消减（→ 主动感知），偶然不确定性是系统内禀随机性，只能容错（→ 鲁棒/概率约束）。U0 原文："把这两者搞混，是新手在选范式时最常见的方向性错误。"

**第二刀：五种安全谱。** 从最保守到最进取：

| | 范式 | 安全定义 | 计算量 | C++ 生态 |
|------|------|---------|--------|---------|
| <span style="color: #e74c3c">最保守</span> | **鲁棒规划** | "对所有扰动都安全" | 低 | 强 |
| | **机会约束** | "违反概率 ≤ δ" | 中 | 弱 |
| | **CVaR** | "最坏 α 尾部平均损失受控" | 中 | 弱 |
| | **POMDP** | "信念空间最大化期望" | 高 | 中 |
| <span style="color: #27ae60">最进取</span> | **分支/场景** | "为各可能未来各备策略" | 中 | 中 |

U0 原文："没有免费的安全——你想要更少的保守度，就得付出更多的建模信息或更多的在线算力。"

**源材料**：U0-U5（30_不确定性规划，7 文件，~12 周）。

---

<h2><span style="color: #27ae60">④</span> 交互是策略性的 —— <span style="color: #27ae60">放松"他人是被动的"</span></h2>

G0 原文："一旦有第二个决策者，'最优'这个词就失去了意义——因为'对我最优'取决于'对方怎么做'，而对方怎么做又取决于'我怎么做'。**这是一个循环，没有单方面的'最优'可言。**"

**三个核心论点：**

**最优 → 均衡。** Nash 均衡刻画的是**稳定性**（没人想单方面偏离），而非最优性。G0 原文："均衡选择是设计决策，不是数学结果。"

**预测后规划 → 预测即均衡。** 传统管线"先预测他人轨迹，再规划避开"在密集交互场景导致 **frozen robot**——预测假设 ego 不动 → 预测出对方不让 → 规划出无法汇入 → ego 真不动，闭环僵死。博弈解法把预测和规划耦合为同一个均衡问题。

**已知代价 → 推断代价。** 对手的代价你不知道——那辆抢行的车是激进型还是保守型？需要从行为反推（逆博弈）。

> <span style="color: #27ae60; font-size: 1.05em"><b>行人突然左转——导航工程师看到"预测错了"（不确定性），自动驾驶研究者追问"他是故意让你先走还是在试探你？"（博弈）。</b></span>

**源材料**：G0-G4（40_博弈规划，6 文件，~10 周）。方法链：HJI 理论 → iLQGames 实时求解 → 逆博弈推断 → CBF+MARL 兜底。

---

<h2><span style="color: #888">⑤</span> 层级抽象 —— <span style="color: #888">管理复杂度的架构模式</span></h2>

这个模型在源材料中没有独立的 Part，但**出现在每一个 Part 的系统架构中**：

| 系统 | 层级链 | 时间尺度 |
|------|--------|---------|
| Apollo | Routing → Scenario → Behavior → Path → Speed → Control | 秒 → ms |
| Nav2 | Global → Behavior → Local → Controller | 任务级 → 10ms |
| PX4 | Task → Global → Trajectory → Tracking | 任务级 → 1ms |

**两个原因**：状态空间爆炸（不能一次性求解全局）、时间尺度不匹配（任务决策秒级、控制执行毫秒级，需要不同工具）。

---

<h2><span style="color: #e2b714">一句 Skeleton Summary</span></h2>

> <span style="color: #e2b714; font-size: 1.05em"><b>移动机器人规划控制 = 在层级架构中求最优决策。当路径与时间强耦合（②）、世界存在不确定性（③）、其他智能体具有策略性（④）时，需要分别引入时空规划、不确定性规划、博弈规划来扩展经典规划框架。</b></span>

三个扩展维度是 T0 原文所说的"三个正交的维度"："不确定性规划放松了'世界是确定的'假设，博弈规划放松了'他人是被动的'假设，时空联合规划放松了'空间时间可分离'假设。三者正交，可以独立学，也常常组合出现。"

MPPI（20_采样式MPC, ~18 周）和多机器人协作（50_多机器人协作, ~21.5 周）在源材料中被定位为**求解方法层**和**系统集成层**——消费三个扩展维度的成果，但不是独立的核心思维模型。
