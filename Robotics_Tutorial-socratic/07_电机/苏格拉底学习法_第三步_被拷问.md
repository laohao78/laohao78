## 第一层：选型与 T-ω（模型 ①②）

**Q1**（<span style="color: #e74c3c">diagnosis</span>）你设计一个新机器人，直接沿用上一代的 M3508 + C620 底盘方案。但新机器人比上一代重了 40%。跑起来之后，电机经常在加速阶段触发 C620 的过流保护，而上一代从没出现过。你用红外测温发现电机外壳也明显更烫。

这不是电机坏了。从 T-ω 曲线和热的角度解释——同一套电机方案为什么在新机器人上"不行了"？

<details>
<summary>点击查看答案</summary>

**负载的 T-ω 需求变大了，但电机的 T-ω 包络线没变。**

机器人重了 40% → 加速需要的转矩大了 40%。电机恒转矩区的上限由电流决定——M3508 的峰值电流是固定的，超过就触发 C620 保护。这不是电机"坏了"，是它被推到了能力边界之外。

更烫的原因也一样：持续电流更大 → 铜损 = I²R 以电流的平方增长（1.4² ≈ 2 倍铜损）→ 温升更快。如果散热条件没变，以前温升 60K，现在可能接近 120K——超过绝缘等级 F 的 115K 温升预算。

**诊断信号**：只在加速阶段触发、电机更烫 → 负载变了，不是硬件故障。

**正确做法**：要么换更大转矩的电机（4010？），要么加减速比。

</details>

**Q2**（<span style="color: #5dade2">cross-connect</span>）你在 SLAM 里学过"传感器和算法要匹配"——16 线雷达配 2D SLAM 是浪费。电机选型里有一个完全类似的匹配逻辑：**电机 T-ω 曲线和负载 T-ω 需求要匹配。**

从 T-ω 曲线的角度解释：为什么一个额定转速 8000 rpm 的小转矩高速电机，不适合直接驱动一个需要 5 Nm 起步转矩的机器人关节？

<details>
<summary>点击查看答案</summary>

**电机的 T-ω 包络线包不住负载的 T-ω 需求。**

负载需要的是"低转速 + 大转矩"——5 Nm 从零速就开始要。高速小转矩电机的 T-ω 曲线在低转速区转矩上限很低（可能只有 0.5 Nm），远低于 5 Nm。

这就是为什么机器人关节电机几乎都配减速器：电机提供"高转速 + 小转矩"，减速器翻译成"低转速 + 大转矩"。功率不变（忽略效率），T × ω 守恒。

这个匹配逻辑和你选传感器的逻辑一样——不是"哪个电机好"，是"哪个电机的 T-ω 曲线包得住你的负载"。

</details>

**Q3**（<span style="color: #e2b714">counterfactual</span>）假设 RoboMaster 组委会突然规定：所有队伍必须用异步电机替代 PMSM。你的云台 yaw 轴现在是一台小型异步电机 + 变频器。

和原来的 GM6020（PMSM + 内置 FOC 驱动器）相比，云台 yaw 轴的低速表现会有什么变化？为什么？

<details>
<summary>点击查看答案</summary>

**低速会变差——异步电机在低速时转子感应不出足够的电流。**

异步电机的转矩依赖"定子磁场比转子快"这个转差。低速时定子磁场转得慢 → 切割导条的速度低 → 感应电流小 → 转子磁场弱 → 转矩小。云台 yaw 轴需要低速平稳跟踪目标，异步电机在这个工况下力不从心。

而 GM6020（PMSM）的转子自带永磁体——转不转都有磁场。零速堵转也能出力，低速转矩密度远高于同体积异步电机。

另外异步电机 + 变频器的控制带宽通常不如 PMSM + FOC 驱动器——云台对转矩脉动极度敏感，异步电机的转矩纹波可能导致画面微抖。

**核心区别**：PMSM 的磁场跟着转子走，异步电机的磁场要"借"——低速时借不到足够的。

</details>

## 第二层：驱动与控制（模型 ③④）

**Q4**（<span style="color: #e74c3c">diagnosis</span>）你调 FOC 电流环。高速段 i_q 跟踪很干净，但转速降到 100 rpm 以下时，i_q 开始出现明显的周期性振荡，电机发出低频嗡嗡声。PI 增益没变，硬件没坏。

最可能的原因是什么？为什么只在低速出现？

<details>
<summary>点击查看答案</summary>

**最可能的原因：死区效应。**

逆变器上下桥臂切换时必须插入死区时间（通常 1-3μs），防止上下管同时导通炸管子。死区期间输出电压为零——实际输出电压比指令电压少了一块。

高速时，调制深度大，死区占比小（3μs / 100μs 周期 ≈ 3%），影响可忽略。低速时调制深度小（比如 10% duty），死区占比暴增（3μs / 10μs ≈ 30%），电压误差相对巨大。

这个误差在电流过零点附近还会发生极性翻转（钳位效应），电流环 PI 拼命补偿 → 过补偿 → 反方向再补 → 振荡。

**诊断信号**：只在低速轻载出现、高频不出现 → 大概率是死区，不是 PI 调得不好。

**正确做法**：做死区补偿（根据电流极性前馈补偿电压），而不是继续调 PI 增益。

</details>

**Q5**（<span style="color: #e2b714">counterfactual</span>）如果你把伺服三环中的速度环去掉，直接用位置环 PI 的输出作为 i_q_ref（跳过速度环），系统还能工作吗？哪些场景会正常，哪些场景会出问题？

<details>
<summary>点击查看答案</summary>

**能工作——但变成了一个"没有阻尼"的位置控制器。**

位置环直接输出 i_q（转矩指令）意味着：角度有偏差 → 直接给转矩。这本质上是一个 PD 控制器（位置环 P 提供比例，编码器微分隐含 D），缺少了速度环的 PI 积分作用。

<span style="color: #27ae60">**正常场景**</span>：恒负载、低速、没有大扰动。位置偏差 → 转矩 → 加速 → 追上 → 减速 → 停，基本能用。

<span style="color: #e74c3c">**出问题的场景**</span>：
- 负载突变（比如云台突然碰到障碍）：少了一层速度环积分 → 掉速后恢复慢，转矩补偿不及时
- 高速运动：速度环的积分项可以提供额外的阻尼——没有它，位置环 P 加大就振荡，减小就响应慢
- 需要精确速度跟踪（如底盘定速巡航）：位置环只关心"到没到"，不关心"跑多快"

**本质**：速度环 = 位置环和电流环之间的缓冲层——把"到哪"翻译成"多快"，再交给电流环执行"多大力"。去掉它，位置环和转矩直接耦合，损失的是对速度的动态控制能力。

</details>

**Q6**（<span style="color: #5dade2">cross-connect</span>）无传感器 FOC 在零速/低速时需要切换到高频注入（HFI）或开环拖动。视觉 SLAM 里的纯 VO 在长距离后也需要回环检测来纠正漂移。

这两组策略——"低速用 HFI / 高速用 BEMF"和"短距离用 VO / 长距离用回环"——在结构上为什么是同一种设计模式？

<details>
<summary>点击查看答案</summary>

**两者都是"信号强时用直接法，信号弱时切到间接法"的多模式切换架构。**

无传感器 FOC：
- 高速段 → BEMF 信号强 → 直接反算角度（直接法，精度高）
- 低速段 → BEMF 信号弱到不可用 → 切到 HFI（间接法，注入信号来"探测"角度）或开环

SLAM：
- 短距离 → 帧间匹配信号强（特征多、视差大）→ 直接算相对位姿（VO，精度高）
- 长距离 → 漂移累积到帧间匹配不可靠 → 切到回环检测（间接法，搜索"我以前来过这里吗"）

**同一个设计原则**：当直接测量信号的信噪比不够时，切换到间接探测策略。两者都不是"一个算法跑到底"——把问题按信号强度分段，每段用不同的原理。

这个模式的代价也一样：切换瞬间如果不平滑（HFI → BEMF 切换时角度跳变 / 回环边加入图优化时位姿跳变），系统会抖一下。

</details>

**Q7**（<span style="color: #e74c3c">diagnosis</span>）你做 RM 云台，上电后 yaw 轴先快速抖动了几下（约 0.5 秒），然后平稳。每次上电都这样。换了电机、换了 C620，现象不变。不是硬件故障。

电流环 PI 增益是调好的，速度环和位置环也正常。抖动只出现在上电后的前 0.5 秒——之后完全正常。最可能的原因是什么？

<details>
<summary>点击查看答案</summary>

**上电时转子初始位置未知，需要"对齐"。**

云台 yaw 轴用的 GM6020 或 M2006 配编码器——但增量编码器上电时不知道绝对角度。驱动器需要做一个对齐流程：给一个已知方向的电流（比如强制拉到 0°），让转子转到这个已知位置，然后以此为原点开始计数。

这个过程就是上电时那 0.5 秒的抖动——转子被电流"拽"到对齐位置。完成后编码器零点建立，FOC 有了准确的角度参考，后续控制就正常了。

如果你用绝对值编码器（断电不丢位置），这个对齐流程可以跳过——上电就不抖。

**诊断信号**：每次都一样、只在上电时、之后完全正常 → 初始化流程，不是控制回路的问题。

</details>

## 第三层：热与系统集成（模型 ⑤ + 跨模型）

**Q8**（<span style="color: #e74c3c">diagnosis</span>）你用一个额定 1kW 的 PMSM 跑一个连续 800W 的负载。电机铭牌上写连续 1kW。前 5 分钟一切正常，第 6 分钟开始转矩逐渐下降，电机外壳烫得手放不上去。

铭牌没造假，负载也只有 800W，低于 1kW 额定。为什么还会出问题？

<details>
<summary>点击查看答案</summary>

**铭牌上的 1kW 是在特定散热条件下标定的——你的散热条件可能达不到。**

可能的原因：
1. **没有强迫风冷**。铭牌上的 1kW 假设有风扇（IC416 风冷），你装在一个密封仓里，热阻大了几倍。
2. **环境温度超过 40°C**。机器人夏天跑，舱内可能 60°C。可用温升从 115K 缩水到 95K。
3. **负载是连续 800W 没错，但铜损是 I²R**。800W 机械功率对应的电流如果已经接近额定电流，铜损在额定散热下是能散的。但如果热阻偏大，同样的损耗下温升更高。

温度升高 → 绕组电阻增大（+0.39%/K）→ 铜损更大 → 温度再升——正反馈。同时 NdFeB 永磁体温度每升 10°C，剩磁下降约 0.1%——转矩常数 k_T 悄悄变小，同样电流出不了同样的力。

**诊断信号**：前几分钟正常、之后逐渐恶化、电机很烫 → 热时间常数问题。热阻偏大或散热不足。

</details>

**Q9**（<span style="color: #5dade2">cross-connect</span>）你在运动控制里学过 MPC 的"模型不完美导致跟踪误差"。电机里有一个完全类似但物理机制不同的现象：**永磁体温度升高 → k_T 下降 → 同样 i_q 出不了同样的转矩。**

这两个"模型不准确导致出力不对"在结构上有什么区别？为什么 MPC 的模型误差通常通过在线估计来补偿，而电机的转矩常数漂移在工程上更多靠温度传感器 + 查表？

<details>
<summary>点击查看答案</summary>

**区别在于模型误差的物理可溯源性。**

MPC 的模型误差来源复杂：未建模动力学（气动、摩擦非线性）、地面交互（软土 vs 硬地）、负载变化（抓取不同物体）。误差维度高、时变、难以直接测量——只能从结果反推（状态估计、扰动观测器）。

电机转矩常数的漂移来源明确且可测量：温度。k_T 随温度下降的规律是已知的（NdFeB ≈ -0.1%/K），而且温度可以在线测量（绕组温度传感器或从电阻变化反算）。所以最简单的方案是"测温度 → 查表修正 k_T"，而不是"从转矩输出偏差反推参数"。

**工程选型原则**：如果误差源已知且可测量 → 前馈补偿（温度查表）。如果误差源复杂且不可直接测量 → 在线估计（扰动观测器/EKF）。电机的温度漂移恰好落在第一类——这是为什么电机的"模型不完美"比足式 MPC 的"模型不完美"好处理得多。

</details>

**Q10**（<span style="color: #e2b714">counterfactual</span>）假设 RM 官方在下一代比赛中禁止使用 FOC 驱动器，只允许用最简单的六步换相 ESC + Hall 传感器来驱动所有电机。

你的云台 yaw/pitch、底盘、拨弹轮各自会受到什么影响？哪些子系统能用六步换相凑合，哪些会直接不达标？

<details>
<summary>点击查看答案</summary>

**拨弹轮** → <span style="color: #27ae60">基本能用</span>。拨弹轮不需要精确转矩控制，恒速转就行。六步换相 + Hall 做速度闭环，够用。

**底盘** → <span style="color: #e2b714">能跑，但体验变差</span>。M3508 轮毂电机在六步换相下转矩脉动会让底盘起步和低速时有微抖动。高速巡航影响不大——换相频率远超人能感知的频段。但需要精确移动（比如对接机关）时，低速抖动会让定位不准。

**云台 yaw/pitch** → <span style="color: #e74c3c; font-weight: bold">直接不达标</span>。云台要求转矩绝对平滑——任何转矩脉动都会直接出现在画面上（微抖动 = 画面糊）。六步换相每 60° 拍一次的对人来说"安静"，对视觉跟踪来说就是周期性扰动。视觉自稳算法可以补偿一部分，但残差很难消除。

**关键认知**：不是六步换相"不好"——是每个子系统对转矩平滑的要求不同。拨弹轮不在乎，底盘有点在乎，云台极度在乎。FOC 的价值不是统一的"更好"，而是对转矩平滑有硬需求的子系统不可替代。

</details>
