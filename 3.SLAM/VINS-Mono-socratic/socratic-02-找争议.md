# VINS-Mono 苏格拉底学习 02：找争议

学习对象：`/root/gpufree-data/VINS-Mono`

核心问题：

> 在单目视觉惯性 SLAM/VIO 里，专家吵得最凶的三个问题是什么？各自站哪边？各自的核心理由是什么？

## 争议 1：滤波式 VIO 还是优化式滑动窗口 VIO？

争议问题：

实时系统应该用 EKF/MSCKF 这类滤波框架，还是用 VINS-Mono 这样的滑动窗口非线性优化？

阵营 A：滤波式方法

核心立场：状态递推快，延迟低，适合算力紧张的平台；很多历史信息通过协方差递推保留下来，不必反复求解一个非线性最小二乘问题。

核心论据：
- 计算量通常更可控。
- 对嵌入式实时系统更友好。
- 实现得当时，短期状态估计足够稳定。

阵营 B：优化式滑动窗口方法

核心立场：在一个局部窗口里反复线性化和联合优化，可以更好处理非线性几何、特征深度、外参、偏置和鲁棒核；代价是需要边缘化和时间预算管理。

核心论据：
- VINS-Mono 在 `Estimator::optimization()` 里同时优化 pose、speed-bias、逆深度、外参和时间偏移。
- Ceres 问题中同时有 IMU 因子、投影因子、边缘化先验和重定位因子，表达能力比单纯递推强。
- `max_solver_time`、`max_num_iterations`、`WINDOW_SIZE` 和边缘化机制把优化问题限制在实时范围内。

底层分歧：

这是“线性化历史是否可以被固定地递推”与“是否值得为更一致的局部重线性化付出计算代价”的方法论分歧。

关联第一步模型：
- 可观测性模型
- IMU 预积分模型
- 滑动窗口因子图模型
- 有界记忆与全局校正模型

拷问你：

如果你说“优化式一定更准”，为什么？在什么场景下这个结论会被实时性、初始化失败或错误边缘化打脸？

## 争议 2：外参、时间偏移和滚快门应离线标定，还是在线估计？

争议问题：

系统应该要求用户提供精确的相机-IMU 外参和同步，还是像 VINS-Mono 一样支持 `estimate_extrinsic`、`estimate_td`、`rolling_shutter` 等在线处理？

阵营 A：离线精确标定优先

核心立场：外参、时间偏移、滚快门读出时间如果先验准确，运行时状态维度更低，优化问题更稳定，也更不容易把运动误差解释成标定误差。

核心论据：
- 在线估计需要足够运动激励，否则不可观测或弱可观测。
- 多估一个变量，就多一个误差吸收通道。
- 工程部署时，稳定的硬件同步和离线标定能显著降低失败模式。

阵营 B：在线估计增强部署鲁棒性

核心立场：真实设备常常没有完美标定和同步；系统如果不能在线修正外参、时间偏移和滚快门效应，很难在用户自制硬件上工作。

核心论据：
- README 明确支持在线外参、时间标定和滚快门。
- `euroc_config.yaml` 里 `estimate_extrinsic` 有 0/1/2 三种模式。
- `Estimator::processImage()` 在 `ESTIMATE_EXTRINSIC == 2` 时要求开始阶段旋转运动，用相邻帧视觉对应和 IMU 预积分初始化外参旋转。
- `ProjectionTdFactor` 把时间偏移 `td` 放入视觉残差。

底层分歧：

这是“可控实验室硬件假设”与“真实部署不完美假设”的分歧。在线估计不是免费午餐，它用更高维度的优化问题换取更低部署门槛。

关联第一步模型：
- 可观测性模型
- 前端测量模型
- 滑动窗口因子图模型

拷问你：

如果初始化阶段没有足够旋转，却打开 `estimate_extrinsic: 2`，你预计系统会卡在哪里？这个失败应归因于优化器、前端，还是可观测性？

## 争议 3：局部 VIO 漂移要不要强反馈全局回环校正？

争议问题：

检测到回环后，应该强行把全局 pose graph 校正反馈进前端/局部 VIO，还是保持局部 VIO 实时运行，只在全局层做漂移校正和可视化路径修正？

阵营 A：强耦合全局一致

核心立场：回环约束应该尽可能影响当前估计，避免局部估计和全局地图长期不一致。

核心论据：
- 全局一致性强，地图复用和多 session 融合更直接。
- 机器人导航时，全局路径和局部状态如果分裂，会带来控制接口复杂性。
- 回环信息如果只做后处理，当前局部状态可能仍沿着漂移方向走。

阵营 B：局部实时优先，全局异步校正

核心立场：VIO 的主任务是低延迟状态估计；回环是慢速、可能误检的全局约束，应通过 pose graph 异步处理漂移，而不应破坏局部估计链路的实时性。

核心论据：
- `PoseGraph::optimize4DoF()` 在独立线程中异步优化。
- pose graph 主要优化 yaw 和平移，保留重力方向约束。
- `Estimator` 里仍然用滑动窗口持续输出局部状态，回环通过 relocalization 信息和 drift correction 影响外部结果。
- DBoW 回环检测有阈值和几何验证，说明回环本身是不确定信息，不能无条件信任。

底层分歧：

这是“全局一致性优先”与“实时局部可控优先”的价值取舍。VINS-Mono 的选择更偏向：局部 VIO 稳定运行，pose graph 作为异步漂移修正层。

关联第一步模型：
- 前端测量模型
- 滑动窗口因子图模型
- 有界记忆与全局校正模型

拷问你：

为什么 pose graph 可以只做 4DoF 优化？如果 IMU 重力方向估计本身错了，4DoF 回环校正会掩盖问题，还是暴露问题？

## 三个争议的共同底层问题

这三个争议表面上分别谈滤波/优化、离线/在线标定、局部/全局耦合，本质都在问同一件事：

> 一个实时 VIO 系统应该把不确定性留在哪里：留在传感器先验里，留在局部优化里，还是留到全局图优化里？

现在反问你：

这三个争议里，你最站哪边？为什么？请不要只说“更鲁棒”或“更实时”，你要指出你愿意牺牲哪个变量：精度、延迟、可部署性、可观测性，还是实现复杂度。
